# PGAGI Analytics Dashboard

A modern, responsive analytics dashboard built with Next.js and Tailwind CSS. It provides insightful data visualizations and interactive components to monitor and analyze key metrics effectively.

##  Project Overview

The PGAGI Analytics Dashboard is designed to offer users a seamless experience in tracking and visualizing data metrics. With a focus on performance and user experience, the dashboard integrates various components to display data in an organized and interactive manner.

##  Installation Instructions

1. **Clone the Repository:**

   ```bash
   git clone https://github.com/latha0001/PGAGI-Analytics-Dashboard.git
   cd PGAGI-Analytics-Dashboard
   ```


2. **Install Dependencies:**

   ```bash
   npm install
   ```

##  How to Run the Project

### Development Server

Start the development server with:

```bash
npm run dev
```

The application will be available at `http://localhost:3000`.

### Production Build

To build the application for production:

```bash
npm run build
```
Then, to start the production server:

```bash
npm start
```

##  Environment Variables

The application requires the following environment variables:

- `API_BASE_URL`: The base URL for the API endpoints.
- `NEXT_PUBLIC_ANALYTICS_ID`: Public analytics tracking ID.

Ensure these are set in the `.env.local` file before running the application.

##  API Setup

If the application integrates with external APIs, obtain the necessary API keys and configure them as environment variables in the `.env.local` file. Ensure that these keys are kept secure and are not committed to version control.


##  Additional Notes or Features

- The project structure follows Next.js conventions, with pages located in the `app` directory.
- Reusable components are organized within the `components` directory.
- Custom hooks can be found in the `hooks` directory.
- Utility functions and libraries are stored in the `lib` directory.
